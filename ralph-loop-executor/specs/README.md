# Specs

Place your feature specification files here. Ralph reads these each iteration.

## Naming Convention

```
specs/
├── 01-overview.md       # what we are building and why
├── 02-data-model.md     # schema, types, data structures
├── 03-api.md            # endpoints or interface contracts
├── 04-tests.md          # what passing looks like
```

## Tips

- Be specific. Vague specs produce vague code.
- Define the completion criteria in `04-tests.md` so Ralph knows when it is done.
- Keep each spec file focused on one concern.
