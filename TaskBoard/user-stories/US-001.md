## Description

Bootstraps the React application, seeds the static question bank with real JEE Mathematics PYQs, and wires the data through to a rendered MCQ card — establishing the full path from raw data to visible UI.

## What to build

**Data layer:** Create a static `questions.json` file containing at least 15 JEE PYQ Mathematics MCQs. Each question object must have: `id` (string), `subject` ("Mathematics"), `question` (string), `options` (`{ A, B, C, D }` object), `correctOption` ("A" | "B" | "C" | "D").

**QuestionBank module:** A pure function `loadQuestions()` that imports `questions.json` and returns a Fisher-Yates shuffled copy. No side effects, no network calls.

**QuestionCard component:** Renders a single question object. Full-screen mobile-first card showing the question text at the top and four labelled option buttons (A, B, C, D) below. No tap interaction yet — options are rendered but non-interactive. Layout must be viewport-height, no horizontal scroll on a 390px-wide screen.

**App wiring:** `App.tsx` calls `loadQuestions()` on mount, holds the shuffled array in state, and passes `questions[0]` to `QuestionCard`. The first card is visible immediately on load — no loading spinner, no splash screen.

**Tests (QuestionBank):** Three unit tests — (1) shuffle produces a different order than the original array at least 9 out of 10 runs (run shuffle 10 times, count mismatches); (2) shuffle returns exactly the same number of questions as the source; (3) each returned object has the required fields `id`, `subject`, `question`, `options`, `correctOption`.

## Acceptance criteria

- [x] `npm run dev` starts the app and shows a JEE question card on a mobile viewport (390 × 844) without any broken layout or horizontal scroll
- [x] The card displays the question text and four option buttons labelled A, B, C, D
- [x] The question shown on each page load is drawn from the shuffled bank (different question may appear on reload)
- [x] `questions.json` contains at least 15 Mathematics questions
- [x] `npm test` runs the QuestionBank unit tests and all three pass
