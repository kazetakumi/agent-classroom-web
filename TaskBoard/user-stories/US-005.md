# US-005: Results Screen and Full Session Lifecycle

**Status:** done
**Blocked By:** US-004
**Parent FEAT:** [Agentic Classroom — JEE Revision Feed](../feats/FEAT-001.md)

---

## Description

Completes the session lifecycle: after End Session, a clean results screen displays the student's performance stats for that session, with Start Again launching a fresh shuffled session and Done returning to the idle state.

## What to build

**QuestionFeed — session summary:** When `endSession()` is called, compute a `SessionSummary` object: `{ correct: number, wrong: number, skipped: number, totalAttempted: number, durationSeconds: number }`. Derive counts by iterating over `session: SessionRecord[]` — for each record, look up the corresponding `Question` by `questionId` and compare `selectedOption` against `question.correctOption`: match → correct, mismatch → wrong, no record for a question → skipped. `totalAttempted` = correct + wrong + skipped. `durationSeconds` = seconds elapsed between the first answer/skip and `endSession()`. Expose the summary via a `summary` field on QuestionFeed state (null while status is not `'ended'`).

**ResultsScreen component:** Pure display component. Props: `summary: SessionSummary`, `onStartAgain: () => void`, `onDone: () => void`. Layout: title ("Session Complete"), three stat blocks — Correct (green), Wrong (red), Skipped (grey) — each showing the count prominently, then total attempted and formatted duration (e.g. "3m 42s"). Two full-width buttons at the bottom: "Start Again" (calls `onStartAgain`) and "Done" (calls `onDone`). Mobile-first layout, same viewport constraints as QuestionCard.

**App wiring:** When `QuestionFeed.status === 'ended'`, render `<ResultsScreen summary={feed.summary} ... />`. `onStartAgain` resets QuestionFeed to a fresh `'active'` session with a re-shuffled question bank and index 0. `onDone` resets QuestionFeed to `'idle'` status — renders a "Start session" landing screen (a single centered button: "Start Revision").

**Idle state / entry screen:** Add `status: 'idle'` as the initial state. App renders a minimal full-screen start screen with the app name and a "Start Revision" button. Tapping it initialises QuestionFeed to `'active'` and loads the shuffled question bank. This completes the full state-machine loop: `idle → active → ended → idle`.

**Tests (ResultsScreen):** (1) correct/wrong/skipped counts render from the `summary` prop values; (2) `onStartAgain` callback fires when "Start Again" is pressed; (3) `onDone` callback fires when "Done" is pressed. **Tests (QuestionFeed):** (1) `summary.correct` counts `SessionRecord` entries where `selectedOption === question.correctOption`; (2) `summary.wrong` counts entries where `selectedOption !== question.correctOption`; (3) `summary.skipped` counts questions in the bank that have no `SessionRecord`; (4) `summary.totalAttempted` equals the sum of the three counts.

## Acceptance criteria

- [x] Ending a session renders the results screen with correct, wrong, and skipped counts matching the answers recorded during the session
- [x] Session duration is displayed in a human-readable format (e.g. "2m 15s")
- [x] "Start Again" immediately begins a new session with a freshly shuffled question bank (first card visible, no intermediate screens)
- [x] "Done" returns to the idle start screen with the "Start Revision" button
- [x] The full state machine loop works: idle → active → ended → idle → active
- [x] All ResultsScreen and QuestionFeed summary unit tests pass
