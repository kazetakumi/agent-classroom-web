# US-008: "Review Questions" Button + Detailed Results Grid

**Status:** done
**Blocked By:** US-007
**Parent FEAT:** FEAT-002 — Detailed Results View — ticket grid and per-question explanation screen

---

## Description

Adds a "Review Questions" button to the session summary (ResultsScreen) and implements the detailed results grid that shows one colour-coded tile per attended question, letting students scan which questions they got right, wrong, or skipped before diving into individual explanations.

## What to build

**ResultsScreen — "Review Questions" button:** Add an `onReviewQuestions: () => void` prop. Render a "Review Questions" button (below the stats, above the Start Again / Done buttons) that calls `onReviewQuestions` when pressed.

**ReviewGrid component:** New component at `src/components/ReviewGrid.tsx`. Props:
- `session: SessionRecord[]` — raw session records from the feed (may contain duplicates; deduplicate by last entry per questionId, matching computeSummary logic)
- `questions: Question[]` — the full question list for subject lookup
- `onBack: () => void` — returns to the summary view
- `onOpenExplanation: (questionId: string) => void` — opens the explanation for a question (implemented in US-009)

Layout: title ("Review Questions"), a scrollable grid of tiles (one per unique attended question), and a "Back" button at the bottom. Each tile:
- `data-testid="result-tile-{questionId}"`
- Background colour: green (`#d1fae5`) for correct, red (`#fee2e2`) for wrong, grey (`#f3f4f6`) for skipped
- Displays the question's `subject` and an outcome label ("Correct" / "Wrong" / "Skipped")
- Tapping calls `onOpenExplanation(questionId)`

**App.tsx wiring:** When `feed.status === 'ended'`:
- `feed.view === 'summary'` → render `<ResultsScreen>` with `onReviewQuestions={feed.openReview}`
- `feed.view === 'review'` → render `<ReviewGrid>` with session/questions/callbacks
- `feed.view === 'explanation'` → render `null` (placeholder; replaced by US-009)

## Acceptance criteria

- [x] ResultsScreen renders a "Review Questions" button
- [x] Clicking "Review Questions" calls `onReviewQuestions`
- [x] ReviewGrid renders one tile per unique attended question (deduplicated by questionId)
- [x] Each tile displays the question subject
- [x] Each tile is colour-coded: green for correct, red for wrong, grey for skipped
- [x] Each tile has `data-testid="result-tile-{questionId}"`
- [x] Clicking a tile calls `onOpenExplanation` with the correct questionId
- [x] ReviewGrid has a "Back" button that calls `onBack`
- [x] App.tsx routes to ReviewGrid when `feed.view === 'review'` and `feed.status === 'ended'`

## Out of scope

- The explanation screen content (US-009)
- Any changes to useQuestionFeed (US-007 already provides openReview / openExplanation / closeExplanation)
