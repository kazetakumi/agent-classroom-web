# US-009: Explanation Screen — Per-Question Answer Highlighting and Explanation Text

**Status:** done
**Blocked By:** US-006, US-007, US-008
**Parent FEAT:** FEAT-002 — Detailed Results View — ticket grid and per-question explanation screen

---

## Description

Implements the per-question explanation screen that a student lands on after tapping a tile in the ReviewGrid. The screen shows the full question, highlights which option the student selected and which option is correct, then walks them through the solution step-by-step using the explanations data loaded by US-006.

## What to build

**ExplanationScreen component** — new component at `src/components/ExplanationScreen.tsx`. Props:
- `question: Question` — the question being explained
- `explanation: Explanation` — the step-by-step explanation from `loadExplanations()`
- `selectedOption: string` — the option the student chose (`'A'`, `'B'`, `'C'`, `'D'`, or `'skipped'`)
- `onBack: () => void` — returns to the ReviewGrid (`closeExplanation()` in App.tsx)

Layout:
- Heading: question subject
- Question text
- All four options (A–D) rendered as non-interactive elements, each with `data-testid="option-{key}"`. Colour-coded via `data-state`:
  - `data-state="correct"` on the correct option (always)
  - `data-state="wrong"` on the student's selected option if it was wrong
  - No `data-state` attribute on options that are neither correct nor the wrong pick
- Explanation steps rendered as a numbered or bulleted list — one element per step string
- "Back" button at the bottom that calls `onBack`

**App.tsx wiring:** Replace the `return null` placeholder (for `feed.view === 'explanation'`) with:
```tsx
if (feed.view === 'explanation') {
  const selectedQuestion = feed.questions.find(q => q.id === feed.selectedQuestionId)!
  const explanations = loadExplanations()
  const explanation = explanations[feed.selectedQuestionId!]
  const sessionRecord = feed.session.filter(r => r.questionId === feed.selectedQuestionId).at(-1)
  return (
    <ExplanationScreen
      question={selectedQuestion}
      explanation={explanation}
      selectedOption={sessionRecord?.selectedOption ?? 'skipped'}
      onBack={() => feed.closeExplanation()}
    />
  )
}
```

## Acceptance criteria

- [x] ExplanationScreen renders the question's subject as a heading
- [x] ExplanationScreen renders the question text
- [x] ExplanationScreen renders all four option texts (A, B, C, D)
- [x] The correct option has `data-state="correct"`
- [x] When the student selected the wrong option, that option has `data-state="wrong"`
- [x] When the student selected the correct option, no option has `data-state="wrong"`
- [x] When the session record is `'skipped'`, only the correct option has `data-state="correct"`; no option has `data-state="wrong"`
- [x] Unselected, non-correct options have no `data-state` attribute
- [x] All explanation steps are rendered
- [x] A "Back" button calls `onBack`
- [x] App.tsx renders ExplanationScreen when `feed.view === 'explanation'` and `feed.status === 'ended'`

## Out of scope

- Any editing or re-answering of questions in the explanation view
- Navigation to next/previous explanations from within the explanation screen
- Any changes to useQuestionFeed, ReviewGrid, or explanations data
