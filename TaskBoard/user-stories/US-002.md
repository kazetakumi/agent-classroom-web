# US-002: Answer Tap, Selection Feedback, and Auto-Advance

**Status:** done
**Blocked By:** US-001
**Parent FEAT:** [Agentic Classroom — JEE Revision Feed](../feats/FEAT-001.md)

---

## Description

Makes the MCQ card interactive: tapping an answer option shows a neutral selection highlight, then automatically advances the feed to the next question — no right or wrong is revealed. The core loop feels like a swipe feed where you simply pick and move on.

## What to build

**QuestionCard interaction:** Add `onAnswer(selectedOption: string)` prop. When an option button is tapped: highlight the tapped button with a neutral selected state (e.g. filled/bold, not green or red). Lock all options (no further taps). After exactly 1.5 seconds, call `onAdvance()` callback. All selection state is local to QuestionCard and resets when a new question is passed in via props.

**QuestionFeed module:** Manages the ordered question list and current session state. State shape: `{ questions: Question[], currentIndex: number, session: SessionRecord[], status: 'active' }`. `SessionRecord` is `{ questionId: string, selectedOption: string }`. Every answered question is appended to `session` — this array is the source of truth for the results screen. Exposes: `currentQuestion` (derived), `submitAnswer(selectedOption)` (appends a `SessionRecord` and triggers advance after QuestionCard's delay). The `submitAnswer` call increments `currentIndex` once QuestionCard fires `onAdvance`.

**App wiring:** Replace static `questions[0]` rendering with `QuestionFeed` state. `App.tsx` holds a `QuestionFeed` instance, passes `currentQuestion` to `QuestionCard`, and pipes `QuestionCard.onAdvance` → `QuestionFeed.advance()`. The feed advances to `questions[1]`, `questions[2]`, etc. on each answer.

**Tests (QuestionCard):** (1) Tapping an option puts that button into a `data-state="selected"` state; (2) No other option changes state when one is tapped; (3) `onAdvance` is called after ~1.5s following a tap (use fake timers). **Tests (QuestionFeed):** (1) `currentIndex` increments by 1 after `advance()` is called; (2) `submitAnswer` appends a `SessionRecord` with the correct `questionId` and `selectedOption`; (3) `session` accumulates one entry per answered question across multiple advances.

## Acceptance criteria

- [x] Tapping an option highlights it with a neutral selected style (no green or red)
- [x] All options are locked after a tap (no double-submit)
- [x] The next question card appears automatically ~1.5 seconds after tapping an answer
- [x] Each answered question is appended to `session` as `{ questionId, selectedOption }`
- [x] `session` persists across advances and is available for the results screen
- [x] No right/wrong feedback is shown at any point during the session
- [x] All QuestionCard and QuestionFeed unit tests pass
