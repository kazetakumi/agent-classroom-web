# US-002: Answer Tap, Feedback Animation, and Auto-Advance

## Description

Makes the MCQ card interactive: tapping an answer option triggers a correct/wrong colour feedback moment, then automatically advances the feed to the next question — delivering the core loop that makes the app feel like a swipe feed.

## What to build

**QuestionCard interaction:** Add `onAnswer(selectedOption: string)` prop. When an option button is tapped: (a) if correct, highlight the tapped button green; (b) if wrong, highlight the tapped button red and highlight the correct option green. Lock all options (no further taps). After exactly 1.5 seconds, call `onAdvance()` callback. All feedback state is local to QuestionCard and resets when a new question is passed in via props.

**QuestionFeed module:** Manages the ordered question list and current session state. State shape: `{ questions: Question[], currentIndex: number, answers: Record<string, 'correct' | 'wrong'>, status: 'active' }`. Exposes: `currentQuestion` (derived), `submitAnswer(selectedOption)` (marks answer, triggers advance after QuestionCard's delay). The `submitAnswer` call increments `currentIndex` once QuestionCard fires `onAdvance`.

**App wiring:** Replace static `questions[0]` rendering with `QuestionFeed` state. `App.tsx` holds a `QuestionFeed` instance, passes `currentQuestion` to `QuestionCard`, and pipes `QuestionCard.onAdvance` → `QuestionFeed.advance()`. The feed advances to `questions[1]`, `questions[2]`, etc. on each answer.

**Tests (QuestionCard):** (1) Tapping the correct option puts the button into a `data-state="correct"` state; (2) Tapping a wrong option puts the tapped button into `data-state="wrong"` and the correct button into `data-state="correct"`; (3) `onAdvance` is called after ~1.5s following a tap (use fake timers). **Tests (QuestionFeed):** (1) `currentIndex` increments by 1 after `advance()` is called; (2) an answered question's result is recorded in `answers` with the correct key; (3) calling `submitAnswer` with the correct option records `'correct'`, wrong option records `'wrong'`.

## Acceptance criteria

- [ ] Tapping a correct answer highlights it green; tapping a wrong answer highlights it red and shows the correct answer in green
- [ ] All options are locked after a tap (no double-submit)
- [ ] The next question card appears automatically ~1.5 seconds after tapping an answer
- [ ] Answers are tracked in QuestionFeed state (correct vs. wrong per question ID)
- [ ] All QuestionCard and QuestionFeed unit tests pass
