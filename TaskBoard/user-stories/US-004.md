# US-004: CompanionSheet — Pull-Up Chat with Pause and End Session

**Status:** todo

**Parent FEAT:** [Agentic Classroom — JEE Revision Feed](../feats/FEAT-001.md)

## Description

Adds the pull-up companion sheet that gives students access to session controls without leaving the quiz flow. Swipe up reveals a partial bottom-sheet overlay with Pause and End Session commands; the question card remains visible beneath it.

## What to build

**SwipeLayer — swipe-up wiring:** Connect the previously-stubbed `onOpenCompanion` callback. Swipe-up gesture (vertical delta > 60px upward) calls `onOpenCompanion`. SwipeLayer itself stays stateless.

**CompanionSheet component:** A bottom-sheet that slides up from the bottom of the viewport. Props: `isOpen: boolean`, `commands: Array<{ label: string; onSelect: () => void }>`, `onClose: () => void`. Renders as a semi-transparent overlay; the question card is still visible in the upper portion of the screen. Each command is a large tap target button. Tapping outside the sheet (the overlay backdrop) calls `onClose`. Swipe-down on the sheet also calls `onClose`. Entrance/exit uses a CSS slide-up/slide-down animation (200ms).

**Session commands wired in App:** Two commands are passed to CompanionSheet:
- **Pause**: calls `QuestionFeed.pause()`. While paused, the question card is dimmed and no gestures are processed. The CompanionSheet is replaced by a "Paused — tap to resume" single-button sheet. Tapping Resume calls `QuestionFeed.resume()` and closes the sheet.
- **End Session**: calls `QuestionFeed.endSession()` which transitions status to `'ended'`. The sheet closes. The app immediately auto-redirects to the ResultsScreen — no intermediate stub or delay. ResultsScreen itself is built in US-005; App.tsx must already route to it when `status === 'ended'` so the transition works end-to-end once US-005 lands.

**QuestionFeed — pause/resume/endSession:** Add `pause()` → sets `status: 'paused'`. Add `resume()` → sets `status: 'active'`. Add `endSession()` → sets `status: 'ended'`, records session end timestamp for duration calculation. Guard skip/goBack/submitAnswer: all are no-ops when status is not `'active'`.

**Tests (CompanionSheet):** (1) renders command buttons for each entry in the `commands` prop; (2) `onClose` is called when the backdrop is clicked; (3) the sheet is not in the DOM when `isOpen` is false (or has `aria-hidden`). **Tests (QuestionFeed):** (1) `pause()` sets status to `'paused'`; (2) `resume()` sets status back to `'active'`; (3) `skip()` is a no-op when status is `'paused'`; (4) `endSession()` sets status to `'ended'`.

## Acceptance criteria

- [ ] Swiping up from the question card opens the companion sheet as a partial overlay (question visible above)
- [ ] The sheet shows "Pause" and "End Session" as tappable commands
- [ ] Tapping "Pause" dims the card and shows a resume prompt; tapping resume returns to active session
- [ ] Tapping "End Session" closes the sheet and immediately auto-redirects to the ResultsScreen (no stub, no intermediate screen)
- [ ] Tapping outside the sheet (backdrop) or swiping down dismisses it without triggering any session command
- [ ] Gestures (skip, go back) are blocked while the session is paused
- [ ] All CompanionSheet and QuestionFeed unit tests pass
