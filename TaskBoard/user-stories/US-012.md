## What to build

Create `OnboardingName.tsx` — the first-entry name collection screen. Column grid layout with two Sage turns, a flex spacer, and an italic Cormorant input for the user's first name. Submits on Return key or → arrow button click. On submit, saves `localStorage.userName` and navigates to Welcome.

Add an entry guard in `App.tsx` (or a new `AppRouter.tsx`): read `localStorage.getItem("userName")` at boot — `null` renders `OnboardingName`, any value renders `WelcomeScreen`. No Sage strip on this screen.

## Acceptance criteria

- [x] First-time users (no `localStorage.userName`) see `OnboardingName` screen
- [x] Returning users (existing `localStorage.userName`) skip to `WelcomeScreen`
- [x] Submitting an empty name is a no-op (no navigation, no storage write)
- [x] Submitting a non-empty name trims whitespace, saves to `localStorage.userName`, and navigates to Welcome
- [x] Return key submits the form
- [x] → arrow button is visually dark/active only when the input has content; dimmed/inactive when empty
- [x] Arrow button click submits when field has content

## Tests

- Empty submit is a no-op — assert `onSubmit` not called, no navigation
- Non-empty submit calls `onSubmit` with trimmed value
- Return key triggers submit
- Arrow darkens only when field has content (check rendered class or style attribute)
