## What to build

Create `ExplanationScreen.tsx` — a four-field full-width layout with labeled sections: QUESTION, SELECTED, CORRECT, EXPLANATION. Selected answer renders dimmed (italic, `color: #C0C0C0`); correct answer renders in full ink (`color: #111`). If `selectedAnswer` is null (skipped question), the SELECTED field shows an em dash. `SageTrigger` with `showNav={true}` at the bottom for ← PREV / progress / NEXT → navigation through all reviewed questions. Component scoped to `.explanation-screen` CSS namespace.

Props: `question: string`, `selectedAnswer: string | null`, `correctAnswer: string`, `explanation: string`, `questionIndex: number`, `totalQuestions: number`, `onPrev: () => void`, `onNext: () => void`, `onAskSage: () => void`, `canGoPrev: boolean`, `canGoNext: boolean`.

## Acceptance criteria

- [ ] Four sections (QUESTION, SELECTED, CORRECT, EXPLANATION) are rendered with their labels
- [ ] SELECTED field shows an em dash (`—`) when `selectedAnswer` is `null`
- [ ] SELECTED answer text renders in italic, `color: #C0C0C0`
- [ ] CORRECT answer text renders in `color: #111`
- [ ] `SageTrigger` renders with `showNav={true}`
- [ ] `onNext` button is disabled / `pointer-events: none` when `canGoNext={false}`
- [ ] `onPrev` is called when PREV button is clicked
- [ ] Component is scoped under `.explanation-screen` CSS namespace

## Tests

- Renders em dash in SELECTED when `selectedAnswer` is `null`
- `onNext` not callable when `canGoNext={false}` (button disabled or pointer-events none)
- `onPrev` called on PREV button click
- All four section labels are present in the DOM
