## What to build

Rewrite `ReviewGrid.tsx` to the D-Seat Map aesthetic. A 4-column CSS grid where each cell is a coloured circle (`aspect-ratio: 1; border-radius: 50%`) coloured by outcome: `.seat--correct` (green), `.seat--wrong` (red), `.seat--skipped` (grey). An IBM Plex Mono filter strip above the grid with tabs: ALL / WRONG / SKIPPED / CORRECT — filters the visible circles. Tapping a circle calls `onOpenExplanation(questionId)` to navigate to the Explanation screen. `SageTrigger` with `showNav={true}` at the bottom.

## Acceptance criteria

- [x] Circles render in a 4-column grid
- [x] Each circle has the correct outcome class (`.seat--correct`, `.seat--wrong`, `.seat--skipped`)
- [x] Filter tab "WRONG" shows only wrong circles; "SKIPPED" only skipped; "CORRECT" only correct; "ALL" shows all
- [x] Tapping a circle calls `onOpenExplanation` with the correct question ID
- [x] Legend dots are present for each outcome type
- [x] `SageTrigger` renders with `showNav={true}`
- [x] Component is scoped under `.review-grid` CSS namespace

## Tests

- Filtering by "WRONG" leaves only `.seat--wrong` circles visible
- Tapping a seat calls `onOpenExplanation(questionId)` with correct ID
- Legend dots for all three outcomes are in the DOM
