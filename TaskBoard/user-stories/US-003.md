# US-003: Swipe Gestures — Skip Forward, Go Back, and Infinite Loop

**Status:** done
**Blocked By:** US-002
**Parent FEAT:** [Agentic Classroom — JEE Revision Feed](../feats/FEAT-001.md)

---

## Description

Adds touch-gesture navigation to the feed: swipe right skips the current question, swipe left returns to the previous card, and the feed loops back to the beginning when the question bank is exhausted — making the experience truly infinite.

## What to build

**SwipeLayer component:** A wrapper component that accepts `onSkip`, `onGoBack`, and `onOpenCompanion` callbacks. Uses pointer/touch events (or a lightweight gesture library such as `react-swipeable`) to detect: swipe-right (horizontal delta > 80px, rightward) → calls `onSkip`; swipe-left (horizontal delta > 80px, leftward) → calls `onGoBack`; swipe-up (vertical delta > 60px, upward) → calls `onOpenCompanion` (wired in US-004; emit but ignore for now). SwipeLayer renders its children unchanged — it is purely a gesture event emitter with no visual output.

**QuestionFeed — skip and go-back:** Add `skip()` action: records the current question as `'skipped'` in the answers map and increments `currentIndex`. Add `goBack()` action: decrements `currentIndex` by 1 (floor at 0; no-op if already at first card). Add `status: 'active'` guard: both actions are no-ops if session is not active.

**Infinite loop:** When `currentIndex` would exceed the last index after either an answer-advance or a skip, reset `currentIndex` to 0 and re-shuffle `questions` via `QuestionBank.loadQuestions()` before rendering the next card. The student never sees a terminal state.

**App wiring:** Wrap `QuestionCard` with `SwipeLayer`. Wire `SwipeLayer.onSkip` → `QuestionFeed.skip()`; `SwipeLayer.onGoBack` → `QuestionFeed.goBack()`.

**Tests (QuestionFeed):** (1) `skip()` records `'skipped'` for the current question and increments the index; (2) `goBack()` decrements the index; (3) `goBack()` at index 0 stays at 0 (no negative index); (4) advancing past the last question wraps back to index 0 with a re-shuffled array (verify array length unchanged, verify same question IDs present).

## Acceptance criteria

- [x] Swiping right on a card skips it and advances to the next question without marking it answered
- [x] Swiping left returns to the immediately previous card (including previously answered or skipped cards)
- [x] Swiping left on the very first card of a session does not crash or navigate to a negative index
- [x] After exhausting all questions the feed silently loops (re-shuffled) with no "end" screen
- [x] All four QuestionFeed unit tests for skip/back/loop pass

## Blocked by

US-002 — provides QuestionFeed module (session state, current index tracking) and SwipeLayer's host component (QuestionCard inside the live feed) needed to attach gestures.
