## What to build

Create `SageTrigger.tsx` — a shared bottom-zone component used by every screen. It stacks two rows: a 44 px nav row (`border-top: 1px solid #F0F0F0`) with ← PREV / progress bar / NEXT → controls, and a 64 px Sage strip (`background: #F9F9F7; border-top: 1px solid #E8E8E0`) labelled "Ask me anything…". The `showNav` prop hides the nav row when `false` (Results screen usage).

Props: `onAskSage`, `onPrev`, `onNext`, `canGoPrev: boolean`, `canGoNext: boolean`, `progress: number` (0–1), `showNav: boolean`.

## Acceptance criteria

- [x] `SageTrigger.tsx` renders both rows when `showNav={true}`
- [x] `SageTrigger.tsx` renders only the Sage strip when `showNav={false}` (nav row absent from DOM)
- [x] PREV button has `pointer-events: none` (or is disabled) when `canGoPrev={false}`
- [x] NEXT button has `pointer-events: none` (or is disabled) when `canGoNext={false}`
- [x] Progress bar width visually reflects the `progress` prop value (0–1 → 0%–100%)
- [x] `onPrev` and `onNext` are called on respective button clicks
- [x] `onAskSage` is called when the Sage strip is tapped
- [x] Component is placed in `src/components/SageTrigger.tsx`

## Tests

Using React Testing Library + Vitest (same pattern as existing tests):

- Renders only Sage strip when `showNav={false}` — assert nav row is not in document
- Renders both rows when `showNav={true}` — assert nav row is present
- `onPrev` called on PREV click; `onNext` called on NEXT click
- PREV has `pointer-events: none` style when `canGoPrev={false}`
- Progress bar width attribute reflects `progress` prop
