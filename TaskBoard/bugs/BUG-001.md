# BUG-001 — Swipe gestures broken on iPhone Chrome

## Summary

Swipe gestures (skip forward, go back, open companion) do not respond when the app is loaded in Chrome on iPhone. The app works correctly on desktop browsers.

## Environment

- **Device:** iPhone (iOS)
- **Browser:** Chrome (WebKit engine — iOS Chrome is WKWebView-based, not Blink)
- **Affected component:** `agent-classroom/src/components/SwipeLayer.tsx`

## Root Cause Analysis

`SwipeLayer` uses React's Pointer Events API (`onPointerDown`, `onPointerUp`, `onPointerCancel`) with `setPointerCapture`. On iOS (both Safari and Chrome, which share the WKWebView/WebKit engine), two compounding issues arise:

1. **`setPointerCapture` unreliable on iOS WebKit** — calling `e.currentTarget.setPointerCapture(e.pointerId)` on a touch pointer can throw or silently fail on iOS, which may prevent `pointerup` from firing on the capturing element.
2. **iOS scroll interference** — even with `touchAction: 'none'`, iOS WebKit sometimes routes the touch sequence to its native scroll handler before the pointer events resolve, resulting in `pointercancel` instead of `pointerup`.

## Steps to Reproduce

1. Open the app URL in Chrome on iPhone.
2. Swipe right on a question card (expected: skip to next question).
3. Swipe left (expected: go back).
4. Swipe up (expected: open CompanionSheet).
5. **Actual:** No gesture is recognised; cards remain static.

## Acceptance Criteria

- [ ] Swipe right (dx > 80) advances to the next question on iPhone Chrome.
- [ ] Swipe left (dx < -80) goes back to the previous question on iPhone Chrome.
- [ ] Swipe up (dy < -60) opens the CompanionSheet on iPhone Chrome.
- [ ] Existing desktop pointer-event behaviour is unchanged.
- [ ] No new scroll-jank introduced on desktop or Android.

## Suggested Fix

Replace (or augment) Pointer Events with `onTouchStart` / `onTouchEnd` fallback handlers:

```tsx
// Use Touch Events as primary on iOS; keep Pointer Events for desktop
function handleTouchStart(e: React.TouchEvent) {
  const t = e.touches[0]
  startPos.current = { x: t.clientX, y: t.clientY }
}

function handleTouchEnd(e: React.TouchEvent) {
  if (!startPos.current) return
  const t = e.changedTouches[0]
  const dx = t.clientX - startPos.current.x
  const dy = t.clientY - startPos.current.y
  startPos.current = null
  // same threshold logic …
}
```

Remove the `setPointerCapture` call (it is not needed if touch events are handled directly) and keep `touchAction: 'none'` on the wrapper `<div>`.
