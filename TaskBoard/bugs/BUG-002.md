# BUG-002 — Results totals always equal full question bank size (17)

## Summary

The Results screen always shows a total of 17 (the full question bank size) regardless of how many questions the user actually attended. Correct, Wrong, and Skipped counts are similarly wrong — questions the user never reached are silently counted as "skipped".

## Environment

- **Affected component:** `agent-classroom/src/questionFeed/useQuestionFeed.ts` — `computeSummary`
- **Visible in:** `agent-classroom/src/components/ResultsScreen.tsx` — `totalAttempted`, `correct`, `wrong`, `skipped` stats

## Root Cause

`computeSummary` iterates over the full `questions` array (all 17 questions) and marks any question with no session record as `skipped`:

```ts
// useQuestionFeed.ts:56-65
for (const question of questions) {          // ← ALL 17 questions
  const record = recordMap.get(question.id)
  if (!record) {
    skipped++                                 // ← never-seen questions counted as skipped
  } else if (record.selectedOption === question.correctOption) {
    correct++
  } else {
    wrong++
  }
}
return { correct, wrong, skipped, totalAttempted: correct + wrong + skipped, ... }
// totalAttempted is always 17
```

## Expected Behaviour

Stats should reflect only questions the user actually interacted with (answered or explicitly swiped past):

| Stat | Definition |
|------|-----------|
| `totalAttempted` | Number of unique questions in `session` records |
| `correct` | Session records where `selectedOption === question.correctOption` |
| `wrong` | Session records where `selectedOption !== correctOption` and `!== 'skipped'` |
| `skipped` | Session records where `selectedOption === 'skipped'` |

Questions never reached are not counted at all.

## Steps to Reproduce

1. Start a session.
2. Answer 3–4 questions, then tap **End Session**.
3. Results screen shows Total = 17, Skipped = 17 − (correct + wrong).

## Acceptance Criteria

- [ ] `totalAttempted` equals the number of questions the user actually interacted with.
- [ ] `correct` counts only answered-correctly questions from the session.
- [ ] `wrong` counts only answered-incorrectly questions from the session.
- [ ] `skipped` counts only questions the user explicitly swiped past (recorded with `selectedOption: 'skipped'`).
- [ ] Ending a session immediately after starting shows all zeros (not 17 skipped).
- [ ] Existing tests pass; new unit test added for `computeSummary` covering a partial session.

## Suggested Fix

Replace the `questions` loop with an iteration over `recordMap.values()`:

```ts
function computeSummary(
  questions: Question[],
  session: SessionRecord[],
  sessionStartTime: number | null,
  endTime: number,
): SessionSummary {
  const questionMap = new Map(questions.map((q) => [q.id, q]))
  const recordMap = new Map(session.map((r) => [r.questionId, r]))
  let correct = 0
  let wrong = 0
  let skipped = 0

  for (const record of recordMap.values()) {
    if (record.selectedOption === 'skipped') {
      skipped++
    } else {
      const question = questionMap.get(record.questionId)
      if (question && record.selectedOption === question.correctOption) correct++
      else wrong++
    }
  }

  const durationSeconds = sessionStartTime
    ? Math.round((endTime - sessionStartTime) / 1000)
    : 0
  return { correct, wrong, skipped, totalAttempted: recordMap.size, durationSeconds }
}
```
