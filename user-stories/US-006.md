# US-006: Explanations data file

## Parent

FEAT-002 — Detailed Results View

## What to build

Create a static `explanations.json` data file keyed by question ID. Each entry contains a detailed, step-by-step solution written in plain, easy-to-understand language. Steps must walk through the maths from first principles — no steps skipped, no jargon left unexplained — so a student who got the question wrong can follow the reasoning and understand *why* the correct answer is right. The file must cover every question in the existing question bank so that the ExplanationScreen never renders a missing-explanation state.

The question bank and explanations file remain decoupled — the ExplanationScreen looks up explanation text by question ID at render time.

## Acceptance criteria

- [x] `explanations.json` (or equivalent static file) exists and is importable by the app
- [x] Every question ID present in the question bank has a corresponding entry
- [x] Each explanation is a numbered step-by-step solution (not a single prose paragraph)
- [x] Steps are written in plain language — no unexplained jargon, no skipped algebra
- [x] A student who got the question wrong can follow the explanation and arrive at the correct answer
- [x] The file format is self-documenting enough that adding new questions is unambiguous
