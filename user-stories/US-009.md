# US-009: Explanation screen

## Parent

FEAT-002 — Detailed Results View

## What to build

End-to-end slice delivering the per-question explanation screen and completing the full review navigation flow.

**ExplanationScreen** is a stateless display component that receives: the question object, the student's selected option (or `null` if skipped), the correct option, the explanation text (looked up from the explanations data file by question ID), and an `onBack` callback.

It renders:
- Full question text
- All four answer options
- The student's selected answer highlighted red (if wrong) — or no highlight if skipped
- The correct answer highlighted green in all cases
- Written explanation text below the options
- A back button that returns to the detailed results grid

Tile tap in `DetailedResultsScreen` calls `openExplanation(questionId)`; the back button on `ExplanationScreen` calls `closeExplanation()`. No inter-question navigation arrows.

Component tests cover three scenarios: correct answer, wrong answer, and skipped — asserting colour highlights and explanation text presence in each case.

## Acceptance criteria

- [ ] Tapping any tile on the grid navigates to the explanation screen for that question
- [ ] Explanation screen shows the full question text
- [ ] All four answer options are displayed
- [ ] Student's wrong selection is highlighted red; no red highlight when question was skipped
- [ ] Correct answer is highlighted green in all scenarios (correct, wrong, skipped)
- [ ] Written explanation text is displayed below the options
- [ ] Back button returns to the detailed results grid (not the summary)
- [ ] Component tests pass for correct, wrong, and skipped scenarios

## Blocked by

- US-006 (explanations data file must exist for lookup at render time)
- US-007 (state machine must expose `openExplanation`, `closeExplanation`, `selectedQuestionId`)
- US-008 (detailed results grid must exist as the navigation origin and destination)
