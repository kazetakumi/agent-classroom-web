# US-007: State machine extension (`useQuestionFeed`)

## Parent

FEAT-002 — Detailed Results View

**Status:** done

## What to build

Extend the `ended` state of `useQuestionFeed` with two new fields and three new actions that drive navigation between the summary, detailed results grid, and explanation screens.

New state fields on `ended`:
- `view: 'summary' | 'review' | 'explanation'` — defaults to `'summary'` when a session ends
- `selectedQuestionId: string | null` — populated when the student taps a tile, `null` otherwise

New actions:
- `openReview()` — transitions `view` to `'review'`
- `openExplanation(questionId: string)` — transitions `view` to `'explanation'` and sets `selectedQuestionId`
- `closeExplanation()` — transitions `view` back to `'review'` and clears `selectedQuestionId`

`startAgain()` and `returnToIdle()` must reset both fields to their defaults.

Unit tests cover every new transition and confirm `selectedQuestionId` is populated/cleared correctly.

## Acceptance criteria

- [x] `view` field exists on the `ended` state and defaults to `'summary'`
- [x] `selectedQuestionId` field exists on the `ended` state and defaults to `null`
- [x] `openReview()` sets `view` to `'review'`
- [x] `openExplanation(id)` sets `view` to `'explanation'` and `selectedQuestionId` to `id`
- [x] `closeExplanation()` sets `view` back to `'review'` and `selectedQuestionId` to `null`
- [x] `startAgain()` and `returnToIdle()` reset both new fields
- [x] All new transitions covered by unit tests following the existing `useQuestionFeed.test.ts` pattern
