# US-008: "Review Questions" button + Detailed Results grid

## Parent

FEAT-002 — Detailed Results View

## What to build

End-to-end slice delivering the entry point into the detailed review flow and the seat-map grid screen.

**ResultsScreen** gains a "Review Questions" button that calls `onReview` (wired to `openReview()` in the state machine). The existing stat cards and duration display are preserved unchanged.

**DetailedResultsScreen** renders a CSS grid where each attended question is a numbered tile coloured by outcome:

| Outcome | Background | Text |
|---------|-----------|------|
| Correct | `#d1fae5` | `#065f46` |
| Wrong | `#fee2e2` | `#991b1b` |
| Skipped | `#fef3c7` | `#92400e` |

Tiles are ordered by session encounter sequence (from `SessionRecord[]`), numbered 1-based. Tapping a tile calls `onSelectQuestion(questionId)`. A back button calls `onBack` (wired to return to the summary view).

Component tests assert: correct tile count, correct colour per outcome, `onSelectQuestion` fires with the right ID on tile click.

## Acceptance criteria

- [ ] "Review Questions" button appears on the summary results screen
- [ ] Tapping "Review Questions" navigates to the detailed results grid
- [ ] Grid shows one tile per attended question (no tiles for unattended questions)
- [ ] Each tile displays its 1-based session position number
- [ ] Correct answers show green background/text
- [ ] Wrong answers show red background/text
- [ ] Skipped answers show amber background/text
- [ ] Tiles are ordered by session encounter sequence
- [ ] Tapping a tile fires the correct callback with the question ID
- [ ] Back button on the grid returns to the summary screen
- [ ] Component tests pass (tile count, colours, callback)
